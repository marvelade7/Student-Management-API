🎓 Student API

Student Management API

Welcome to the Student Management GraphQL API documentation. This API allows you to manage students, departments, and faculties.

Live API Endpoint

https://student-management-api-v9qe.onrender.com/graphql

Note: All requests must be made via HTTP POST to the endpoint shown above.

Create a Student

Registers a new student onto the platform. Requires personal details, department ID, and a unique matriculation number.

Request Body Parameters

FIELD TYPE REQUIRED DESCRIPTION
firstName string YES Student's first name
lastName string YES Student's last name
email string YES Valid unique email address
gender string YES Gender of the student
department string (ID) YES Valid Department Object ID
level int YES Current academic level (e.g., 100)
age int YES Age of the student
matricNumber string YES Unique matriculation number 
mutation {
  createStudent(
    input: {
      firstName: "John"
      lastName: "Doe"
      email: "[EMAIL_ADDRESS]"
      gender: "Male"
      department: "69234accc8f3db7deaf59631"
      level: 300
      age: 20
      matricNumber: "CSC/2026/001"
    }
  ) {
    id
    firstName
    lastName
  }
}
{
  "data": {
    "createStudent": {
      "id": "64f2a1b9...",
      "firstName": "John",
      "lastName": "Doe"
    }
  }
}

Get All Students

Retrieves a list of all students in the database along with their associated department details.

Request Body Parameters

No parameters required for this query.

query {
  students {
    id
    firstName
    email
    department {
      name
    }
  }
}
{
  "data": {
    "students": [
      {
        "id": "64f2a1b9...",
        "firstName": "John",
        "email": "john@example.com",
        "department": {
          "name": "Computer Science"
        }
      }
    ]
  }
}

Update Student

Updates an existing student's details. You only need to send the fields you wish to update.

Request Body Parameters

FIELD TYPE REQUIRED DESCRIPTION
id string (ID) YES The ID of the student to update
input object YES Object containing the fields to update
input.firstName string NO Student's new first name
input.level int NO Student's new level 
mutation {
  updateStudent(
    id: "64f2a1b9...",
    input: {
      level: 400
    }
  ) {
    id
    firstName
    level
  }
}
{
  "data": {
    "updateStudent": {
      "id": "64f2a1b9...",
      "firstName": "John",
      "level": 400
    }
  }
}

Delete Student

Removes a student from the database entirely.

Request Body Parameters

FIELD TYPE REQUIRED DESCRIPTION
id string (ID) YES The ID of the student to delete 
mutation {
  deleteStudent(id: "64f2a1b9...") {
    id
    firstName
  }
}
{
  "data": {
    "deleteStudent": {
      "id": "64f2a1b9...",
      "firstName": "John"
    }
  }
}

Create Department

Creates a new department under a specific faculty.

Request Body Parameters

FIELD TYPE REQUIRED DESCRIPTION
name string YES Department name
faculty string (ID) YES Faculty ID this department belongs to 
mutation {
  createDepartment(
    input: {
      name: "Computer Science"
      faculty: "64e9c1d..."
    }
  ) {
    id
    name
  }
}
{
  "data": {
    "createDepartment": {
      "id": "64f1b2c3d...",
      "name": "Computer Science"
    }
  }
}

Get All Departments

Fetches all departments and their associated faculty.

query {
  departments {
    id
    name
    faculty {
      name
    }
  }
}
{
  "data": {
    "departments": [
      {
        "id": "64f1b2c3d...",
        "name": "Computer Science",
        "faculty": {
          "name": "Faculty of Science"
        }
      }
    ]
  }
}
